KOALA supports a pure S3 mode and a S3 mode with more comfort. The pure S3 mode presents the contend of the buckets exactly. S3 is a completely flat name space. Therefore S3 has no support for folders. However, folders can be simulated with clever use of key names. The S3 mode with more comfort uses keys that end with _$folder$ as directory placeholders. A key that is meant staying inside are folder has a name following this schema folder/subfolder/filename. It is the same approach S3Fox uses. {% endblock %}
